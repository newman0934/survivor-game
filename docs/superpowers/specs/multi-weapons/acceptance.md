# Acceptance — 多種武器 + 武器專屬升級

本檔為本功能完成與否的**唯一驗收來源**。所有項目通過才得標記為完成。
（不重複 BDD scenario；此處為可勾選的驗收清單。）

## 武器共存與開火
- [ ] 起始只持有「魔杖」Lv1，並會自動鎖定最近敵人開火
- [ ] 可透過升級卡解鎖「飛刀」「聖經」「大蒜」
- [ ] 解鎖新武器不會覆蓋既有武器；多把武器同時存在
- [ ] 每把武器各自依自己的冷卻計時器獨立開火
- [ ] 可收齊全部四把武器並同時運作

## 各武器行為正確
- [ ] 魔杖：鎖定最近敵人、單發直線；高等級可鎖定最近 N 隻
- [ ] 飛刀：朝玩家最後非零移動方向發射；多發時並排/扇形
- [ ] 聖經：N 本以玩家為圓心環繞旋轉；碰到敵人扣血
- [ ] 大蒜：玩家周圍圓形場域，範圍內敵人持續扣血（dmg×dt）

## 武器專屬升級（VS 風）
- [ ] 每把武器最高 5 級
- [ ] 升級套用該等級表預定的固定效果（數量/傷害/冷卻/半徑/轉速）
- [ ] 升級卡面正確顯示等級資訊（如「飛刀 Lv2→Lv3」）

## 全域被動（乘區）
- [ ] PlayerStats 改為乘區：damageMult / cooldownMult / projectileSpeedMult / areaMult + moveSpeed + pickupRadius
- [ ] 全域被動同時影響所有武器的生效數值（生效值 = 等級值 × 乘區）
- [ ] 既有 5 張升級卡改寫為乘區被動且行為正確

## 升級卡池
- [ ] 候選動態依當前 World 狀態產生（持有/等級/上限）
- [ ] 三類候選：解鎖新武器 / 升級既有武器 / 全域被動
- [ ] 武器持有達上限 4 後，不再產生「解鎖新武器」候選
- [ ] 武器達 Lv5 後，不再產生該武器的升級候選
- [ ] 候選不足 3 張時以「補血」保底卡補滿至 3 張
- [ ] 每次升級恰好提供 3 張不重複候選

## 邊界與錯誤處理
- [ ] 場上無敵人時，鎖定型武器不開火、無例外、無除以零
- [ ] 玩家從未移動時，飛刀朝預設方向（向右）發射
- [ ] 一次拾取跨多個等級門檻時，每次升級都重抽當前合法候選
- [ ] 套用未知升級 id 時安靜略過、不丟例外
- [ ] 聖經對同一敵人在命中冷卻期間只扣血一次

## 確定性與架構邊界
- [ ] 所有隨機走 seeded rng，無 Math.random()
- [ ] 相同 seed + 相同選擇序列 → 相同升級候選序列
- [ ] systems/weapons.ts、leveling.ts、types.ts 無 Vue/Pinia 執行期 import
- [ ] 固定步長 1/60 不變
- [ ] Game.stop() / PixiRenderer.destroy() 維持冪等
